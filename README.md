* 서기 20XX년, 전 세계의 데이터가 알 수 없는 **'디지털 노이즈'**에 오염되었습니다. 이 사고로 인류의 가장 소중한 자산 중 하나인 **[완벽한 커피 레시피]**들이 모두 삭제되거나 깨진 코드로 변해버렸습니다. 사람들은 커피 맛을 잃어버려 만성 피로에 시달리고 있으며, 세상은 활기를 잃고 잿빛으로 변해갑니다. 이에 정부는 흩어진 데이터 조각을 모아 맛을 복원하는 특수 기관, **[픽셀 카페 레시피 수사국]**을 설립합니다. 이곳의 요원들은 고도로 훈련된 파이썬(혹은 C언어) 기술을 이용해 훼손된 레시피 수치를 입력하고, 시스템을 재구축하여 인류에게 '카페인'을 되찾아줘야 합니다.

* "신입 요원, 시스템에 접속한 것을 환영한다."

당신은 이제 막 수사국에 배치되었습니다. 현재 수사국의 메인 시스템은 초기화되어 요원 정보조차 남아있지 않은 상태입니다.

첫 번째 임무: 시스템에 본인의 **코드네임(이름)**과 보안 등급을 등록하여 접근 권한을 획득하십시오.

두 번째 임무: 수사국이 확보한 첫 번째 **'오염된 레시피 데이터'**의 수치를 정확히 입력하여 분석기에 전달하십시오.

## 🟦 [1차 과제: V1.0] 시나리오 기획
    
### **🤖 AI 파트너십 과정**
 1. **내용 1 : 사이버 펑크, 픽셀 카페 레시피 수사국 컨셉 도출**
    * **프롬프트 요약:** ""파이썬으로 나만의 카페 게임 같은 프로그램을 만들고 싶어. 내가 어떤 역할을 맡아서 이 프로그램을 관리하면 1학년 학생들이 재미있게 코딩할 수 있을까? 멋진 컨셉 하나만 추천해줘.""
    * **적용 내용:** AI가 제안한 여러 아이디어 중 **'카페 총지배인'**이라는 컨셉이 가장 마음에 들어 채택함. 단순히 코드를 짜는 게 아니라 '카페를 운영한다'는 목표를 세우니 기획이 훨씬 즐거워짐.
      
 2. **내용 2 : 로드맵 도출**
    * **프롬프트 요약:** "1차 과제 필수 요건(변수 5개, 자료형 3개 이상)을 충족하기 위한 카페 레시피 초기 스탯 데이터 구조 모델링 논의"
    * **적용 내용:** 포괄적인 의미의 '카페 메뉴 가격' 변수 하나를 사용하는 대신, 데이터 특성에 맞춰 문자형(`char` - 요원의 코드네임), 정수형(`int` - 요원의 보안 등급, 레시피 물의 양, 예상 판매가), 실수형(`float` - 복원할 원두 무게)으로 자료형을 명확히 분리하여 입력 데이터의 구조적 완성도를 높임.

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**

  * 이번 과제는 용사 등록 및 정밀 진단 시스템 구현으로, 사용자로부터 데이터를 입력받아 리스트에 저장하고, 가중치 연산을 통해 결과값을 도출하는 시스템을 구축하는 것입니다. 특히 `if-elif` 조건문과 `and/or` 논리 연산자를 활용하여 상황에 맞는 등급과 나만의 카페를 부여하는 지능형 로직을 완성해야 합니다.
  * input()으로 요원 코드네임, 보안등급, 원두무게, 총 제조량을 입력받아 리스트에 저장.
  * 가중치 공식을 적용하여 요원의 등급 산출: (미션점수 * 0.4) + (성공률 * 100 * 0.5) + (근속연수 * 10)
  * if-elif-else를 사용하여 레시피 복원을 했냐 안했냐를 판단할거야.
  * and 연산자를 활용하여 보안등급과 레시피를 활용해 고급 레시피 복원을 확인할거야.
    
### **🤖 AI 파트너십 과정**
1. **내용 1: 보안 요원 컨셉 및 로직 도출**
    * **프롬프트 요약:** "내가 카페 보안요원이 되어 카페레시피를 관리하는 프로그램을 만들 거야. 1학년 수준에서 리스트와 조건문을 활용해 등급을 매기는 재미있는 로직을 짜줘."
    * **적용 내용:** AI가 제안한 '요원의 미션점수' 방식과 'S등급이면 시니언 요원 칭호 부여'라는 복합 조건 아이디어를 실제 코드에 반영함.

2. **내용 2: 효율적인 리스트 활용법 자문**
    * **프롬프트 요약:** "파이썬 기초 문법 단계(리스트 → 조건문 → 반복문 → 함수/딕셔너리)에 맞춰, '카페 레시피 보안요원' 컨셉의 프로그램이 어떻게 발전할 수 있을지 4단계 기능 확장 로드맵을 제안해줘."
    * **적용 내용:** 단순 입출력을 넘어, **'보안데이터 베이스 구축(1차) → 일괄 보안 스캔(2차) → 일급 비밀 정보함(3차) → 보안 자동화 시스템 (함수)(4차)'**로 이어지는 논리적인 확장 로드맵을 확립함.
    
### **🛠️ Troubleshooting & 기술 회고:**
(예시) 
1. **문제 1: "분명 S급인데 출입 거부가 떠요!" (Logical Error)**
    * **원인:** `대소문자 구분 문제임. 파이썬은 "S"와 "s"를 완전히 다른 데이터로 취급함. 보안 시스템이 너무 깐깐해서 소문자로 말하면 못 알아듣는 것임.
    * **해결:** `.upper() 함수를 사용해 입력값을 무조건 대문자로 변환하거나, if agent_level == "S" or agent_level == "s": 처럼 두 경우를 모두 체크하도록 수정함.
2. **문제 2: 코드가 제멋대로 실행돼요!" (IndentationError)**
    * **원인:** `들여쓰기(Indentation) 실수임. 파이썬은 들여쓰기를 통해 "어디까지가 조건문에 포함되는 코드인가?"를 결정하는데, 이 줄을 맞추지 않으면 보안 구역 설정이 엉망이 됨.
    * **해결:** `if문 바로 아랫줄은 반드시 키보드의 Tab 키나 Space 4번을 눌러서 '이 코드는 내 구역이야!'라고 표시해줘야 한다는 규칙을 적용하여 수정함.
